## General
1. 알고리즘의 효율성
   1. 선형 검색과 이진 검색에서, 이진 검색을 하기 위해서는 정렬이 필요한데, 이 정렬할 때의 시간이 오히려 선형 검색보다 더 걸리면 비효율적. 그래서 그런 것까지 생각할 수 있는 사고 방식이 필요하다.
2. 정렬 알고리즘
   1. 선택 정렬
   2. 거품 정렬
   3. 병합 정렬
   4. 삽입 정렬(팬케이크 정렬)*
3. 알고리즘의 시간 복잡도
   1. 빅 오: 최악의 경우, 알고리즘 실행 시간의 상한
   2. 빅 오메가 : 최선의 경우, 알고리즘 실행 시간의 하한

## Mission Review 
(각 알고리즘에 대한 시간복잡도를 정확히 계산하는 것도 중요했다.)
1. 숫자 애너그램 찾기
    - 정렬을 사용하지 않고 해결
      - 메모리를 더 할당해서
2. 친구 집과 가까운 집으로 이사 가기
   -  절대값들의 합이 최소가 되는 것은 중앙값이 최소가 된다.
   -  특수한 경우에는 시간 복잡도가 더 빠르게 찾을 수 있다.
3. 다리 건너기 (우리팀이 우수제출팀!!)
   - 시간이 '1,3,4,10'인 경우에 기존 알고리즘이 가장 빠른 방법이 아니다!!!
      - 이것 말고 또 예외가 있나?? 그것은 어떻게 찾을 수 있지?
4. 가장 큰 낙하 거리 찾기
   -  각 열의 최상단에 있는 상자에 대해서만 계산하면 된다!
